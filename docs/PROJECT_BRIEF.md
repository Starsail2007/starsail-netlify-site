# Project Brief

## 当前项目是什么

这是星帆 Starsail 的个人静态网站项目。它的主线仍是“安静入口”和“待探索空间”，但项目已经从单一门户页扩展为一个可持续维护的 Astro 多页面站点：保留个人入口体验，同时承载 maimai 成绩页、世界杯实时展示页和若干设计/交互实验页。

项目的未来内容方向仍由用户决定。Codex 的职责是把已经明确的想法整理成可维护的页面、组件、样式、脚本、数据和部署路径，不主动替用户预设未来要放什么。

## 当前页面状态

- 首页：夜间默认的个人入口，包含头像展示、入口选项、更新日志弹层和若干入口/loader 原型。
- maimai：静态快照优先的成绩展示页，包含 B50、Rating 曲线、同步时间线和建议面板；远端刷新是可选能力。
- 世界杯：实时赛事展示页，包含赛程滚轮、日期筛选、分组/淘汰赛、关键瞬间和冠军庆祝层；数据优先读取 GitHub 静态 JSON，并保留 Netlify Function 兜底。
- 实验页：保留入口改版、loader、候选方案等探索页面，作为设计迭代空间。
- 公网主地址：`https://starsail.netlify.app/`
- GitHub Pages 静态备用地址：`https://starsail2007.github.io/starsail-netlify-site/`

## 为什么整理成这个结构

项目已经从单个 HTML 文件变成可持续开发的静态站点骨架。这样之后无论是改文案、改排版、接入 Figma/Canva 的设计稿、维护静态数据，还是继续拆分 maimai/世界杯这类功能系统，都可以在清晰的目录里进行。

静态网页文案集中在 `src/content/site-text.md`。B50、Rating、世界杯赛程/比分、球员名映射等动态或派生内容不作为普通文案维护，而是由数据文件、脚本、第三方接口或 GitHub Actions 生成。

## 给未来 Codex 的提醒

这个项目的“未来要做什么”由用户决定。Codex 应负责把用户的设想转化为可维护的网页结构、组件、样式和交互，不要主动替用户设定内容方向。
